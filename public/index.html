How signals are scored
Open methodology · v0.2.0
⌖ The scoring formula on this page

Cross-source consensus

Each topic is attributed to the sources that contributed at least one document — arXiv, GitHub, Hacker News, Reddit, HuggingFace, Product Hunt, Replicate, Bluesky, Semantic Scholar. The consensus ratio is the fraction of sources that fetched successfully on this snapshot and confirmed the topic.

consensus_ratio = |sources_confirming| / |active_sources|

A topic seen on 4 of 5 active sources renders as the 4/5 ✓ badge. AlphaSignal and TLDR can't do this because they're per-source emails.

Hidden-gem score

hidden_gem = 0.40·velocity + 0.35·(1 − saturation/100) + 0.25·builder_signal
+ 0.20·avg(venue_boost)   # capped at 1.0

venue_boost fires (0.5) when an attributed arXiv paper's arxiv:comment mentions a top-tier venue (ICML / NeurIPS / ICLR / CVPR / EMNLP / ACL / NAACL / AAAI / SIGKDD / ICDM / ECML / COLM) with a 2025–2039 year.

Trend-Before-Trend Score (TBTS)

tbts = 100 · (0.35·velocity + 0.30·hidden_gem + 0.20·lifecycle + 0.15·convergence)

Saturation

saturation = 0.35·github + 0.30·hn + 0.20·arxiv + 0.15·semantic_scholar
  (each input is the topic's percentile against today's top-N)
